# Issue #102 調査結果：GitHubでClaude Codeを管理するといい理由

> 対象読者：Claude Codeを使い始めたばかりの人、Git未経験者
> 記事化の素材として整理したドラフトです。

---

## まず整理：Claude の「種類」を理解する

Claude には大きく3つの使い方があります。

| 名前 | 何をするもの | 向いている人 |
|------|------------|------------|
| **Claude Chat** | 質問・文章作成・調べ物 | 誰でも |
| **Claude Cowork** | ブラウザ・メール・ファイル操作を自動化 | 非エンジニア向けの業務自動化 |
| **Claude Code** | コードを書く・ファイルを操作する・開発作業を自律実行 | 開発者・設定を使いこなしたい人 |

一言で言うと：「**Chat は思考、Cowork は実行、Code は構築**」

---

## Claude Code にも「版」がある

Claude Code は3つの動かし方があります。

| 版 | 起動方法 | 特徴 |
|---|---------|------|
| **Web版** | ブラウザで claude.ai/code を開く | インストール不要。ただしローカルファイルへのアクセスに制限あり |
| **デスクトップアプリ版** | Desktop アプリを起動 | GUIで操作。ビジュアルdiff・複数タスク並行作業が得意 |
| **CLI版** | ターミナルで `claude` コマンドを実行 | 最も自由度が高い。自動化・スクリプト化・フック（hooks）が使える |

**CLIじゃないと使えない機能がある**のが重要なポイントです（後述）。

---

## GitHubで管理するとは？

Claude Codeを使うとき、以下のようなファイルを自分で書きます：

- **CLAUDE.md**：Claudeへの指示書（「いつもタメ語で話して」「mainに直接pushしないで」など）
- **スキル（/コマンド）**：よく使う作業を自動化したもの（例：`/commit` でコミット、`/review` でレビュー）
- **フック（hooks）**：特定のタイミングで自動実行されるスクリプト（例：セッション開始時に自動で最新情報を取得）

これらをGitHubのリポジトリで管理する、というのがここで言う「GitHubで管理する」です。

---

## メリット

### 1. 変更履歴が残る
「前の指示の方が良かった」「いつからこの設定にしたっけ？」が解決します。Gitのコミット履歴をたどれば、いつ・何を・なぜ変えたかがわかります。

### 2. 複数環境で同じ設定を使える
自宅PC・会社PC・クラウドなど、どの環境でも同じCLAUDE.mdとスキルを使えます。`git pull` 一発で最新の設定が同期されます。

### 3. 実例：section9リポジトリ

このリポジトリ自体がその実例です。

```
section9/
├── core.md          ← Claudeへの指示書（チームルール・キャラ定義など）
├── docs/            ← ルール・ナレッジの蓄積
│   └── learnings.md ← セッションをまたいで知見を蓄積
└── automations/     ← スキル・フック・自動化スクリプト
    ├── morning-routine/  ← 毎朝の定例作業を自動化
    ├── discord-9ka/      ← Discordと連携した自動返答
    └── ...（15スキル）
```

- セッションが変わっても `learnings.md` に過去の知見が残るので、同じミスを繰り返さない
- スキルを追加・改善するたびにPRでレビューし、コミット履歴に記録
- フック（SessionStartフック）でセッション開始時に自動で最新設定を読み込む

### 4. 「育てていける」
最初はシンプルなCLAUDE.mdだけで始め、使いながらスキルを追加していけます。GitHubで管理することで、その成長の歴史が記録されます。

## デメリット・注意点

- **Gitの学習コストがある**：コミット・プッシュの概念は最初は慣れが必要
- **CLI版が前提になりやすい**：フックやスキルの一部機能はCLI版でしか動かない（デスクトップアプリ版では未対応、2026/03時点）
- **管理の手間**：ファイルが増えてくると整理が必要になる

## Git未経験者向け：GitとGitHubって何？

### ゲームに例えると

RPGで「セーブ」をするとき、「スロット1」「スロット2」と複数の状態を保存できますよね。Gitはそれをファイルでやるツールです。

- **Gitのセーブ（コミット）**：「この状態を記録しておく」という操作
- **セーブの歴史**：「昨日の版」「先週の版」にいつでも戻れる
- **メッセージ付き**：「ここで〇〇を追加した」と理由も一緒に残せる

GitHubは、そのセーブデータをインターネット上に置けるサービスです。

```
自分のPC（ローカル）       GitHub（ネット上）
  ┌──────────┐             ┌──────────┐
  │ CLAUDE.md │ ──push──▶  │ CLAUDE.md │
  │ スキル群  │            │ スキル群  │
  └──────────┘             └──────────┘
      ↑ 手元で編集                ↑ バックアップ＆共有
```

### 最低限の用語

| 用語 | 一言で言うと | 具体例 |
|------|------------|-------|
| **リポジトリ** | プロジェクトごとのフォルダ（セーブ履歴つき） | `section9` リポジトリ |
| **コミット** | 変更を「セーブ」する操作 | 「CLAUDE.mdにルール追加」とメモして保存 |
| **プッシュ** | ローカルのセーブをGitHubに送る操作 | 手元の変更をネット上に反映 |
| **プル** | GitHubの最新版を手元に取り込む操作 | 別のPCで続きをやるときに使う |
| **ブランチ** | 「試し書き用のコピー」を作る機能 | 本番を壊さずに実験できる |

### Gitを使わないとどうなる？

- CLAUDE.mdを編集したら「前の版」が消える
- 「いつ・何を・なぜ変えたか」がわからなくなる
- 複数のPCで使うたびに手動でファイルをコピーする必要がある
- 誤って大事な設定を上書きしても戻せない

Claude Codeの設定ファイルはコードと同じで「育てていくもの」なので、Gitで管理するのが自然な選択です。

---

## Claude Code を推したい理由（おまけ）

- **自律的に動く**：「○○して」と頼めば、ファイルを読んで・考えて・実行して・確認してくれる
- **指示が蓄積される**：CLAUDE.mdに書いたルールは毎回読まれる。「毎回同じことを説明する手間」がなくなる
- **GitHubと相性がいい**：Claudeにコミット・PR・Issueコメントまでやってもらえる
- **Chat版との違い**：ChatはAIと「会話」するが、Codeは「仕事を任せる」感覚に近い
